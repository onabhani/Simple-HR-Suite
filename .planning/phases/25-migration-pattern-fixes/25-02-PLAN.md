<objective>
Replace all information_schema queries with either `SHOW TABLES LIKE` / `SHOW COLUMNS FROM` (for runtime checks) or version-gated patterns (for migration code).

Purpose: information_schema queries are slow on shared hosting (full metadata scan), run on every admin page load in several places, and couple the plugin to MySQL internals. `SHOW TABLES LIKE` and `SHOW COLUMNS FROM` are faster, cached by MySQL, and the standard WordPress pattern.

Output: Zero information_schema references remain in runtime code paths. Migration helpers use SHOW-based equivalents.
</objective>

Table existence check (replaces information_schema.tables):
```php
// OLD:
$exists = (int)$wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = DATABASE() AND table_name = %s",
    $table
));

// NEW:
$exists = $wpdb->get_var($wpdb->prepare("SHOW TABLES LIKE %s", $table));
// Returns table name string if exists, null if not
// Use: if ($exists) { ... } or if (!$exists) return;
```

Column existence check (replaces information_schema.COLUMNS):
```php
// OLD:
$exists = (int)$wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(*) FROM information_schema.COLUMNS
     WHERE table_schema = DATABASE() AND table_name = %s AND column_name = %s",
    $table, $column
));

// NEW:
$exists = $wpdb->get_var($wpdb->prepare("SHOW COLUMNS FROM `{$table}` LIKE %s", $column));
// Returns column name string if exists, null if not
```

Index existence check (replaces information_schema.STATISTICS):
```php
// OLD:
$exists = (int)$wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(*) FROM information_schema.STATISTICS
     WHERE table_schema = DATABASE() AND table_name = %s AND index_name = %s",
    $table, $index
));

// NEW:
$exists = $wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(*) FROM (SHOW INDEX FROM `{$table}` WHERE Key_name = %s) AS idx",
    $index
));
// Actually SHOW INDEX doesn't support subquery. Use instead:
$indexes = $wpdb->get_results("SHOW INDEX FROM `{$table}`", ARRAY_A);
$exists = false;
foreach ($indexes as $idx) {
    if ($idx['Key_name'] === $index_name) { $exists = true; break; }
}
// OR simpler: just try and suppress error, or use the pattern from WordPress core:
$exists = $wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(1) FROM information_schema.STATISTICS WHERE table_schema = DATABASE() AND table_name = %s AND index_name = %s",
    $table, $index_name
));
// NOTE: For index checks, information_schema.STATISTICS is acceptable in migration-only code
// because there is no SHOW equivalent that supports WHERE on index_name cleanly.
// The requirement says "no information_schema at RUNTIME" — migration-only is OK if version-gated.
```

Column type check (replaces information_schema.COLUMNS for COLUMN_TYPE):
```php
// OLD:
$col_type = $wpdb->get_var($wpdb->prepare(
    "SELECT COLUMN_TYPE FROM information_schema.columns WHERE table_schema = DATABASE() AND table_name = %s AND column_name = %s",
    $table, 'status'
));

// NEW:
$row = $wpdb->get_row($wpdb->prepare("SHOW COLUMNS FROM `{$table}` LIKE %s", 'status'), ARRAY_A);
$col_type = $row ? $row['Type'] : null;
```

Foreign key check (replaces information_schema.TABLE_CONSTRAINTS):
```php
// These only run inside one-time migration (guarded by get_option('sfs_hr_att_fk_migrated')).
// Acceptable to keep information_schema for FK checks in migration-only code since
// SHOW CREATE TABLE parsing is fragile. Ensure it's behind version/option gate.
```

Engine check (replaces information_schema.TABLES for ENGINE):
```php
// OLD:
$engine = $wpdb->get_var($wpdb->prepare(
    "SELECT ENGINE FROM information_schema.TABLES WHERE TABLE_SCHEMA = DATABASE() AND TABLE_NAME = %s", $t
));

// NEW — use SHOW TABLE STATUS:
$row = $wpdb->get_row($wpdb->prepare("SHOW TABLE STATUS LIKE %s", $t), ARRAY_A);
$engine = $row ? $row['Engine'] : null;
```
