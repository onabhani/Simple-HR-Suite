<task type="auto">
  <name>Task 1: Fix unprepared SHOW queries in Frontend tabs and Loans module</name>
  <files>
    includes/Frontend/Tabs/OverviewTab.php,
    includes/Frontend/Tabs/ProfileTab.php,
    includes/Frontend/Tabs/PayslipsTab.php,
    includes/Modules/Loans/LoansModule.php
  </files>
  <action>
Replace every unprepared SHOW TABLES LIKE and SHOW COLUMNS FROM query with $wpdb->prepare() equivalents. Specifically:

**OverviewTab.php** — 3 fixes at lines ~140, ~164, ~205:
- Change `$wpdb->get_var( "SHOW TABLES LIKE '{$loans_table}'" )` to `$wpdb->get_var( $wpdb->prepare( "SHOW TABLES LIKE %s", $loans_table ) )`
- Same pattern for `$sess_table` and `$assign_table`
- Note: The comparison `=== $loans_table` still works because prepare() with SHOW TABLES LIKE returns the table name if found

**ProfileTab.php** — 1 fix at line ~392:
- Change `$wpdb->get_var( "SHOW TABLES LIKE '{$assign_table}'" )` to `$wpdb->get_var( $wpdb->prepare( "SHOW TABLES LIKE %s", $assign_table ) )`

**PayslipsTab.php** — 1 fix at line ~38:
- Change `$wpdb->get_var( "SHOW TABLES LIKE '{$payslips_table}'" )` to `$wpdb->get_var( $wpdb->prepare( "SHOW TABLES LIKE %s", $payslips_table ) )`

**LoansModule.php** — 8 fixes:
- Lines ~64, ~125, ~154: Change 3x `SHOW TABLES LIKE '{$loans_table}'` to use `$wpdb->prepare( "SHOW TABLES LIKE %s", $loans_table )`
- Line ~131: Change `SHOW COLUMNS FROM {$loans_table} LIKE 'cancellation_reason'` to `$wpdb->prepare( "SHOW COLUMNS FROM {$loans_table} LIKE %s", 'cancellation_reason' )`
- Lines ~159, ~173, ~184, ~195, ~206: Change each `SHOW COLUMNS FROM {$loans_table} LIKE 'column_name'` to use `$wpdb->prepare()` with `%s` placeholder for the column name string

For all SHOW COLUMNS FROM fixes: keep the `{$loans_table}` interpolation for the table name (it comes from `$wpdb->prefix` which is safe), but use `%s` placeholder for the LIKE value.

Do NOT change the `manage_options` references in LoansModule.php lines 66 and 90 — those are not in scope for DEBT-02 (which is specifically about Setup_Wizard and Company_Profile only).
  </action>

  </verify>
