**DATA-04: Fix tenure boundary in LeaveCalculationService::compute_quota_for_year()**

In `compute_quota_for_year()` at line 43, the tenure is calculated relative to January 1 of the year:
```php
$as_of = strtotime($year . '-01-01');
```
This means an employee hired on 2022-06-15 would NOT get the 5-year bump until Jan 1 2028, when they should get it on June 15 2027.

Fix: Calculate tenure at the employee's anniversary date within the given year. If the anniversary has not yet occurred in the given year, use the previous year's tenure. Change lines 43-47 to:
```php
// Compute tenure at employee's anniversary date in the given year
$hire_md = substr($hire_date, 5); // MM-DD portion
$anniversary_in_year = $year . '-' . $hire_md;
$as_of = strtotime($anniversary_in_year);
if (!$as_of) {
    // Handle Feb 29 edge case: fall back to Mar 1
    $as_of = strtotime($year . '-03-01');
}
$hd = strtotime($hire_date);
if (!$hd) return $quota;

$years = (int) floor(($as_of - $hd) / (365.2425 * DAY_IN_SECONDS));
if ($years < 0) $years = 0;
```
This ensures the quota steps up on the anniversary, not Jan 1. For example, if the employee was hired 2022-06-15, in year 2027 the anniversary is 2027-06-15 giving exactly 5 years, triggering the ge5 quota.

Update the docblock to note: "Tenure computed at employee's anniversary date within the year, not January 1."
