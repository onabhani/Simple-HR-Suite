  <action>
Read every PHP file listed above. For each file, check against the following audit metrics and record every finding:

**Security (Critical/High) -- PRIMARY FOCUS for this plan:**
- SQL injection: Every `$wpdb->query()`, `$wpdb->get_results()`, `$wpdb->get_var()`, `$wpdb->get_row()` call -- is the query built with `$wpdb->prepare()` or is user input concatenated/interpolated? Flag raw interpolation as Critical.
- REST auth: Every `register_rest_route` callback -- does `permission_callback` check appropriate capability (`sfs_hr.view`, `sfs_hr.manage`, etc.)? Flag `__return_true` or missing permission_callback as Critical.
- AJAX auth: Every `wp_ajax_*` handler in admin pages -- does it verify nonce and check `current_user_can()`? Flag missing checks as High.
- Nonce validation: Every form submission and AJAX callback -- `wp_verify_nonce()` or `check_ajax_referer()` present? Flag missing as High.
- Input sanitization: `$_POST`, `$_GET`, `$_REQUEST`, `$request->get_param()` -- sanitized before use? Flag unsanitized as High.
- Output escaping: Data echoed to HTML -- `esc_html()`, `esc_attr()`, `esc_url()`, `wp_kses()` used? Flag unescaped as Medium. Pay special attention to Frontend shortcodes which render directly to the page.
- Kiosk security: The kiosk shortcode is publicly accessible -- check if it properly validates employee identity, prevents session spoofing, and handles selfie upload securely. Flag issues as High/Critical.

**Performance (Medium/High):**
- N+1 queries in admin reports: Admin pages that list attendance records -- do they query employee details inside a loop? Flag as High.
- Unbounded queries: Report pages that SELECT all sessions/records without LIMIT or date bounds. Flag as Medium.
- Missing pagination: Admin list pages without pagination on large datasets. Flag as Medium.
- Heavy AJAX handlers: Handlers that run expensive queries without caching. Flag as Medium.

**Logical Issues (varies):**
- REST endpoint authorization scope: Can a regular employee access admin-only endpoints? Can employees see other employees' attendance data? Flag as High.
- Early leave approval logic: Can requests be approved/rejected by unauthorized users? State machine transitions correct? Flag as High.
- Kiosk edge cases: What happens if employee punches in twice? If geolocation fails? If selfie upload fails mid-request? Flag as Medium.
- Widget data leakage: Does the widget shortcode show attendance data only for the logged-in employee or could it leak other employees' data? Flag as High if leaking.

For each finding, record:
1. **Severity**: Critical / High / Medium / Low
2. **Category**: Security, Performance, or Logic
3. **File and line number(s)**
4. **Description**: One sentence
5. **Evidence**: Code snippet (1-5 lines)
6. **Fix recommendation**: Concrete fix

IMPORTANT: Start with class-admin-pages.php (6984 lines, most findings expected). Read it in chunks (lines 1-2000, 2001-4000, 4001-6984). Then REST endpoints, then Frontend shortcodes.
