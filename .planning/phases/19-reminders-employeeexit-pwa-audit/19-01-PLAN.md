Read all 7 files completely. Audit against the 4 metrics:

**1. Security (Critical focus -- cron notification escaping and SQL safety):**

RemindersModule.php:
- Check for bare ALTER TABLE, unprepared SHOW TABLES, information_schema queries (antipattern from Phase 04/08/11/12/16/18).
- Check install()/activation hooks for DDL safety.
- What capability is used to gate the admin menu page? Is it appropriate or recycled from another module (like sfs_hr_attendance_view_team pattern from Phase 15/16)?

class-reminders-cron.php:
- Are notification queries unbounded (no LIMIT)? On a large org with 1,000+ employees, does the cron query ALL employees every run?
- Is notification content escaped before output (esc_html, wp_kses) or sent raw into email/HTML?
- Are all $wpdb queries using $wpdb->prepare()?

class-reminders-service.php:
- Are all $wpdb queries using $wpdb->prepare()? Catalogue every raw interpolation.
- Does the service enforce any authorization, or is it pure data access?
- Any unbounded SELECT queries without LIMIT?

class-celebrations-page.php:
- POST handler audit: Does every handler have both nonce verification AND capability check?
- Is output on the celebrations page escaped?

class-dashboard-widget.php:
- Is the widget content escaped?
- Does the widget query respect department scoping or show all-org data?

EmployeeExitModule.php:
- Check for bare ALTER TABLE, unprepared SHOW TABLES, information_schema queries.
- Does this module contain any end-of-service calculation logic? If so, compare with Settlement module (Phase 10 found SETT-LOGIC-001: UAE 21-day formula instead of Saudi 15-day formula). Flag if EmployeeExit copies or references the same incorrect formula.
- What capability gates the admin page?

class-employee-exit-admin.php:
- POST handler audit: Does every handler have both nonce verification AND capability check? (Check for nonce-only guard without capability -- Phase 13 pattern.)
- Are all $wpdb queries using $wpdb->prepare()?
- Exit clearance workflow: Can any authenticated user mark clearance items complete? Is there ownership/role validation?
- Any __return_true REST permission callbacks?

**2. Performance:**
- N+1 query patterns: Does reminder notification query employee data inside loops?
- Unbounded queries: Are there any SELECT without LIMIT on sfs_hr_employees for notification targets?
- Cron efficiency: Does the cron job query all employees every run or use date-targeted queries?
- EmployeeExit: Any heavy queries on admin page load?

**3. Duplication:**
- Does EmployeeExit duplicate end-of-service calculation logic from Settlement module?
- Does Reminders duplicate notification dispatch logic from Core/Notifications?
- Does Reminders duplicate holiday/anniversary date calculation from other modules?
- Any repeated SQL patterns that could use a shared helper?

**4. Logical Issues:**
- Reminder scheduling: Are birthday/anniversary reminders sent on the correct date accounting for timezone?
- EmployeeExit checklist: Can exit items be completed out of order? Is there state machine enforcement?
- EmployeeExit + Resignation interaction: Does EmployeeExit trigger on resignation approval? Is there a race with Settlement module?
- Dead code: Any methods that exist but are never called?
- Race conditions: Any check-then-act patterns without DB-level guards (TOCTOU -- same as Phase 14/17)?
