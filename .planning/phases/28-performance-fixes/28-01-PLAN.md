<objective>
Fix N+1 query patterns and add transient caching to the admin-side dashboard, org chart, analytics, Loans widget, and Reminders widget.

Purpose: The Core admin dashboard is the highest-traffic admin page. It currently fires 13+ uncached queries on every load (CORE-PERF-002), has an N+1 pattern in the org chart (CORE-PERF-003, 2 queries per department), runs 3 aggregate analytics queries uncached (CORE-PERF-004), and the Loans dashboard widget fires 8 uncached queries (LADM-PERF-002). The Reminders widget fires up to 16 N+1 date-targeted queries per render (REM-PERF-001).

Output: Admin dashboard with transient-cached counters, batch org chart rendering, cached analytics, cached Loans widget, batch Reminders widget.
</objective>

  <action>
Three fixes in Admin.php:

**1. Dashboard counter caching (CORE-PERF-002):**
Find the dashboard rendering method (around line 87-270 area) that fires 13+ separate COUNT/SELECT queries for employee counts, leave counts, attendance stats, etc. Wrap ALL these counter queries in a single transient cache:
- Key: `sfs_hr_admin_dashboard_counts`
- TTL: 60 seconds (short — data is operational)
- On cache hit, skip all individual queries and use cached array
- On cache miss, run all queries, store result array in transient
- Do NOT change the query logic itself — just wrap with get_transient/set_transient

**2. Org chart N+1 fix (CORE-PERF-003):**
Find `render_org_chart()` or similar method (around line 4413-4518) that loops over departments calling `get_user_by()` + `$wpdb->get_row()` per department. Fix:
- Before the loop, batch-fetch ALL department managers in one query: `SELECT user_id, display_name FROM {$wpdb->users} WHERE ID IN (...)` using the manager IDs from the departments array
- Before the loop, batch-fetch ALL department employee counts in one query: `SELECT department_id, COUNT(*) as cnt FROM {$emp_table} WHERE status='active' GROUP BY department_id`
- Inside the loop, look up from the pre-fetched arrays instead of querying

**3. Analytics section caching (CORE-PERF-004):**
Find `render_analytics_section()` (around line 256-270 area) that runs 3 aggregate queries on every dashboard load. Wrap in transient:
- Key: `sfs_hr_admin_analytics`
- TTL: 300 seconds (5 minutes — aggregate data changes slowly)
- Same get/set pattern as dashboard counters
