Two fixes:

**1. OverviewTab counter caching (FE-PERF-001):**
In `OverviewTab.php` (lines 59-233 area), the `render()` method fires 9+ separate DB queries for leave balance, attendance status, pending requests, loan counts, etc. Add transient caching:
- Key: `sfs_hr_overview_tab_{$employee_id}` (per-employee cache since data is employee-scoped)
- TTL: 60 seconds
- Wrap the entire block of counter queries: on cache hit, use cached array; on cache miss, run all queries and store
- Ensure the transient is deleted when relevant actions occur (leave approval, attendance punch) — add `delete_transient()` calls to the appropriate action hooks if feasible, otherwise rely on TTL expiry

**2. Role_Resolver per-request cache (FE-PERF-003):**
In `Role_Resolver.php`, the `resolve()` method fires 4+ DB queries on every portal page load. Add a static property cache:
- Add `private static $cache = [];` to the class
- At the start of `resolve()`, check `if (isset(self::$cache[$user_id])) return self::$cache[$user_id];`
- Before returning, store: `self::$cache[$user_id] = $result;`
- This is a per-request cache (static property resets on next PHP request) — no transients needed

Three fixes:

**1. Leave status count queries (LV-API-PERF-001):**
In `LeaveModule.php`, find the `render_requests()` method or its helper that fires 11 separate `SELECT COUNT(*) ... WHERE status = 'X'` queries for tab badges (pending, approved, rejected, cancelled, etc.). Replace with a single query:
```sql
SELECT status, COUNT(*) as cnt FROM {$req_table} [WHERE department scoping] GROUP BY status
```
Then build the counts array from the single result set. Cache with transient key `sfs_hr_leave_counts_{$scoping_key}`, TTL 60 seconds.

**2. Leave approver name N+1 (LV-PERF-003):**
In the same `render_requests()` method, find the loop that calls `get_user_by('id', $approver_id)` per row to resolve approver display names. Fix:
- Before the loop, collect all unique approver IDs from the result set
- Batch-fetch: `SELECT ID, display_name FROM {$wpdb->users} WHERE ID IN (...)`
- Build a lookup array `$approver_names[$id] = $display_name`
- Inside the loop, look up from the array instead of calling get_user_by()

**3. Leave history pagination (LV-API-PERF-003):**
In `employee_tab_content()` or the method that renders leave history for a specific employee, find the unbounded SELECT and add `LIMIT 50`. This is a detail view so 50 most recent is sufficient.

**4. Resignation status counts (RSVC-PERF-001 / RES-PERF-001):**
In `class-resignation-service.php`, find `get_status_counts()` that fires 6 separate COUNT queries. Replace with:
```sql
SELECT status, COUNT(*) as cnt FROM {$res_table} GROUP BY status
```
Also check `class-resignation-admin.php` for any duplicate count pattern (RES-PERF-001) and fix there too if present.
