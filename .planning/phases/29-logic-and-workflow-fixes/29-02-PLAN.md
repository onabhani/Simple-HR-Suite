**Performance review state machine (in Reviews_Service.php):**

The existing code already has per-method guards (line 235: check `submitted` before acknowledge, line 270: check `draft`/`pending` before submit). However, there's no central guard, and other direct `$wpdb->update` calls could bypass these checks.

Add a constant at the top of the `Reviews_Service` class:

```php
private const ALLOWED_TRANSITIONS = [
    'draft'        => [ 'pending', 'submitted' ],
    'pending'      => [ 'submitted' ],
    'submitted'    => [ 'acknowledged' ],
    'acknowledged' => [],   // terminal
];
```

Add a private helper method:

```php
/**
 * Validate a status transition is allowed.
 *
 * @return true|\WP_Error
 */
private static function validate_transition( string $from, string $to ) {
    $allowed = self::ALLOWED_TRANSITIONS[ $from ] ?? [];
    if ( ! in_array( $to, $allowed, true ) ) {
        return new \WP_Error(
            'invalid_transition',
            sprintf( __( 'Cannot change review status from %s to %s.', 'sfs-hr' ), $from, $to )
        );
    }
    return true;
}
```

Update `submit_review()` to use the helper: replace the existing status check (line 270) with:

```php
$check = self::validate_transition( $review->status, 'submitted' );
if ( is_wp_error( $check ) ) {
    return $check;
}
```

Update `acknowledge_review()` similarly: replace the existing status check (line 235) with:

```php
$check = self::validate_transition( $review->status, 'acknowledged' );
if ( is_wp_error( $check ) ) {
    return $check;
}
```

This centralizes the transition logic while preserving the existing return type (\WP_Error).

```

```

Add a private constant near the top of the LeaveModule class (before any methods):

```php
/**
 * Allowed leave request status transitions.

    'pending'                => [ 'approved', 'rejected', 'cancelled', 'pending_gm', 'pending_hr' ],
    'pending_gm'             => [ 'approved', 'rejected', 'pending_hr' ],
    'pending_hr'             => [ 'approved', 'rejected' ],

    'on_leave'               => [ 'returned', 'early_returned' ],

    'returned'               => [],
    'early_returned'         => [],
    'cancellation_rejected'  => [ 'approved' ],
];
```

Add a private helper method:

```php
/**
 * Check if a status transition is allowed.
 *
 * @param string $from Current status
 * @param string $to   Target status
 * @return bool
 */
private function is_valid_transition( string $from, string $to ): bool {
    $allowed = self::ALLOWED_TRANSITIONS[ $from ] ?? [];
    return in_array( $to, $allowed, true );
}
```
