<task type="auto">
  <name>Task 1: Read and audit Payroll module orchestrator and REST endpoints</name>
  <files>
    includes/Modules/Payroll/PayrollModule.php (700 lines -- module bootstrap, payroll run logic, calculation, hook registration)
    includes/Modules/Payroll/Rest/Payroll_Rest.php (263 lines -- REST API endpoints for payroll data and operations)
  </files>
  <action>
Read both PHP files. Audit each file against the 4 metrics:

**1. Security (Critical focus):**
- Every `$wpdb->query()`, `$wpdb->get_results()`, `$wpdb->get_row()`, `$wpdb->get_var()` call: Is `$wpdb->prepare()` used? Flag any raw string interpolation as Critical.
- PayrollModule table installation/migration: Does it use bare `ALTER TABLE` or unprepared `SHOW TABLES` like Loans/Core did? (Prior phases found this as Critical antipattern -- LOAN-SEC-001/002, Phase 04 Critical.)
- REST endpoint permission callbacks: Does every route have a proper `permission_callback` that checks capabilities (e.g., `sfs_hr.manage`)? Flag `__return_true` as Critical (same pattern as Attendance Phase 05).
- Payroll run trigger: Is there a capability check before running payroll? Can a non-admin trigger a payroll run via REST or admin_post?
- Nonce validation: Are all write operations (run payroll, update payroll record) protected with nonce checks?

**2. Performance:**
- Payroll run: Does it load all employees at once or paginate? For a company with 500+ employees, is this an unbounded query?
- N+1 patterns: When calculating payroll for each employee, does it make per-employee sub-queries for salary, deductions, loans, attendance? Count the query-per-employee pattern.
- Are payroll results cached or recalculated on every view?

**3. Duplication:**
- Cross-module: Does PayrollModule recalculate loan deductions independently from the Loans module? (LOAN-LOGIC-002 from Phase 08 found PayrollModule references `l.monthly_installment` which does not exist -- verify this finding and check for other schema mismatches.)
- Does payroll calculation duplicate salary component logic that exists in Employee/Settlement modules?
- Are there repeated date-range calculations (month boundaries, working days) that duplicate Core helpers?

**4. Logical Issues:**
- Floating-point rounding: Are salary calculations using PHP float arithmetic without rounding? Check for `round()` usage on financial amounts. Flag missing rounding as High.
- Zero-day periods: What happens if an employee has zero working days in a payroll period (e.g., hired on last day, on unpaid leave entire month)?
- Mid-month hires: Is salary prorated correctly for employees who start mid-month? What formula is used?
- Payroll re-run: Can payroll be run twice for the same period? Does it overwrite or duplicate records?
- Negative amounts: Can deductions exceed gross salary, resulting in negative net pay? Is there a floor?
- Payroll period boundaries: How are month start/end dates determined? Is timezone handling consistent?

Rate every finding: Critical / High / Medium / Low. Include file:line references and a concrete fix recommendation for each.
