### Integration Issues (3)

**HIGH — handle_reject() missing exit (LOGIC-02)**

`LeaveModule.php` line 1801: `is_valid_transition()` guard calls `wp_safe_redirect()` without `exit;`. All other 5 handlers have proper termination. Execution falls through on invalid transition, allowing a duplicate rejection sequence.

**MEDIUM — handle_approve() balance not transaction-wrapped (LOGIC-01/DATA-02)**

Phase 27's read-before-write balance pattern in `handle_approve()` (lines 1706-1760) runs outside any transaction. Phase 29 transactions protect creation paths only. Concurrent dual-approval for the same employee can race on balance updates.

**LOW — No cache invalidation after status mutations (PERF-03)**

Phase 28's `sfs_hr_leave_counts_*` and `sfs_hr_admin_dashboard_counts` transients (60s TTL) have no `delete_transient` at mutation sites. Phase 29 adds 6 guarded mutation paths — none bust the cache. Display staleness up to 60s after status changes.
