**File 1: class-resignation-handlers.php**

In `handle_submit()` around line 77, replace the unvalidated redirect:
```php
$redirect_url = isset($_POST['_wp_http_referer']) ? $_POST['_wp_http_referer'] : admin_url('admin.php?page=sfs-hr-my-profile');
```
with a validated one:
```php
$default_url = admin_url('admin.php?page=sfs-hr-my-profile');
$redirect_url = isset($_POST['_wp_http_referer'])
    ? wp_validate_redirect( wp_unslash( $_POST['_wp_http_referer'] ), $default_url )
    : $default_url;
```
`wp_validate_redirect()` checks the URL against `allowed_redirect_hosts` filter and the current site host. Any external URL falls back to `$default_url`. The `wp_unslash()` is needed because WordPress slashes POST data.

Do NOT change anything else in handle_submit or any other handler.

**File 2: class-resignation-list.php**

In `render()` around lines 24-27, add an `sfs_hr.view` capability check for non-manage users AND enforce empty result when a manager has no departments. Replace the current scoping block:
```php
// Department scoping for non-HR managers
$dept_ids = [];
if (!current_user_can('sfs_hr.manage')) {
    $dept_ids = Resignation_Service::get_manager_dept_ids(get_current_user_id());
}
```
with:
```php
// Department scoping for non-HR managers
$dept_ids = [];
if (!current_user_can('sfs_hr.manage')) {
    if (!current_user_can('sfs_hr.view')) {
        echo '<p>' . esc_html__('You do not have permission to view resignations.', 'sfs-hr') . '</p>';
        return;
    }
    $dept_ids = Resignation_Service::get_manager_dept_ids(get_current_user_id());
    if (empty($dept_ids)) {
        echo '<p>' . esc_html__('No resignations found for your departments.', 'sfs-hr') . '</p>';
        return;
    }
}
```
This ensures:
1. Non-manage users must have `sfs_hr.view` (granted dynamically to department managers)
2. If a manager has no departments assigned, they see a clean message, not all resignations
3. The `$dept_ids` array is always non-empty when used, so the service correctly scopes the query
