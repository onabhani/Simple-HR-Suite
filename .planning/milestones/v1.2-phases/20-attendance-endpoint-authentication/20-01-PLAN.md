<objective>
Fix all four attendance authentication vulnerabilities: gate the /status and /verify-pin REST endpoints behind login checks, enforce nonce-before-capability ordering on the admin rebuild handler, and replace plain SHA-256 token hashes in kiosk roster with HMAC-SHA-256 using a per-roster rotating nonce.

Purpose: Eliminate unauthenticated access to attendance data and prevent offline punch forgery via exposed token hashes.
Output: Hardened attendance endpoints with no public access and HMAC-based offline kiosk validation.
</objective>

**Server side — `class-attendance-rest.php`, `kiosk_roster()` method (lines 265-334):**

**PWA IndexedDB — `pwa-app.js`:**

5. Add an `hmacSha256` helper function next to the existing `sha256` helper (around line 199):
   ```javascript
   window.sfsHrPwa.hmacSha256 = async function(message, key) {
       var enc = new TextEncoder();
       var keyData = await crypto.subtle.importKey(
           'raw', enc.encode(key), { name: 'HMAC', hash: 'SHA-256' }, false, ['sign']
       );
       var sig = await crypto.subtle.sign('HMAC', keyData, enc.encode(message));
       return Array.from(new Uint8Array(sig)).map(function(b) {
           return b.toString(16).padStart(2, '0');
       }).join('');
   };
   ```
