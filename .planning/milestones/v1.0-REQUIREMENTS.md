# Requirements: Attendance Module Refactor — Phases 1–3

**Defined:** 2026-03-09
**Core Value:** AttendanceModule.php becomes a clean orchestrator under 500 lines that delegates to focused classes

## v1 Requirements

Requirements for this refactor. Each maps to roadmap phases.

### Views Extraction

- [x] **VIEW-01**: Extract shortcode_widget() (~1900 lines) into `Frontend/Widget_Shortcode.php` with a `render()` method
- [x] **VIEW-02**: Extract shortcode_kiosk() (~2800 lines) into `Frontend/Kiosk_Shortcode.php` with a `render()` method
- [x] **VIEW-03**: All inline JS/CSS preserved as-is in view output — no asset file extraction
- [x] **VIEW-04**: AttendanceModule shortcode methods become thin delegates to Frontend classes

### Migration Extraction

- [x] **MIGR-01**: Extract maybe_install() (~300 lines) into module-local `Migration.php`
- [x] **MIGR-02**: Migration class follows existing `CREATE TABLE IF NOT EXISTS` + `add_column_if_missing()` pattern
- [x] **MIGR-03**: AttendanceModule calls Migration class from its existing hook

### Service Extraction

- [x] **SVC-01**: Extract period logic into `Services/Period_Service.php`
- [x] **SVC-02**: Extract shift resolution into `Services/Shift_Service.php`
- [x] **SVC-03**: Extract session recalc into `Services/Session_Service.php`
- [x] **SVC-04**: Extract early leave logic into `Services/Early_Leave_Service.php`
- [x] **SVC-05**: AttendanceModule delegates to service classes for backwards compatibility

### Cleanup

- [x] **CLEN-01**: Remove or consolidate remaining helper/delegate methods that are no longer needed
- [x] **CLEN-02**: AttendanceModule.php reduced to under 500 lines (achieved: 434 lines after Phase 3; intermediate PR showed 5,390 lines before service extraction)
- [x] **CLEN-03**: Zero behavior change — all existing functionality preserved
