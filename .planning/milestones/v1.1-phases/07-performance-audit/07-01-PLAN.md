<task type="auto">
  <name>Task 1: Read and audit Performance services, calculator, and cron</name>
  <files>
    includes/Modules/Performance/Services/Performance_Calculator.php (415 lines -- scoring formulas, weighted aggregation)
    includes/Modules/Performance/Services/Reviews_Service.php (460 lines -- review CRUD, submission, status management)
    includes/Modules/Performance/Services/Goals_Service.php (388 lines -- goals CRUD, goal-based scoring)
    includes/Modules/Performance/Services/Attendance_Metrics.php (520 lines -- attendance data integration for performance scoring)
    includes/Modules/Performance/Services/Alerts_Service.php (706 lines -- alert rules, notification triggers)
    includes/Modules/Performance/Cron/Performance_Cron.php (976 lines -- scheduled review cycles, reminder cron, alert evaluation)
  </files>
  <action>
Read all 6 PHP files. Audit each file against the 4 metrics:

**1. Security (Critical focus):**
- Every `$wpdb->query()`, `$wpdb->get_results()`, `$wpdb->get_row()`, `$wpdb->get_var()` call: Is `$wpdb->prepare()` used? Flag any raw string interpolation as Critical.
- Do service methods validate caller capabilities before performing write operations (create/update/delete reviews, goals, alerts)?
- Are nonce checks present where these services are called from user-facing actions?
- Can an employee read their own performance score before it is published/finalized? Check access control on score retrieval methods.

**2. Performance:**
- N+1 query patterns: Does Attendance_Metrics.php query per-employee in a loop when it could batch?
- Unbounded queries: Do cron jobs or alert evaluations query all employees without LIMIT? Flag if cron runs on every employee without pagination.
- Are there expensive JOINs or subqueries in scoring aggregation that would degrade on large employee sets?
- Does Performance_Cron schedule overlap checks (multiple cron instances running simultaneously)?

**3. Duplication:**
- Is scoring/aggregation logic repeated between Performance_Calculator.php and Reviews_Service.php?
- Do Goals_Service.php and Reviews_Service.php duplicate CRUD patterns that could share a base helper?
- Is date range handling for review periods duplicated across files?
- Are there repeated attendance query patterns between Attendance_Metrics.php and the Attendance module itself?

**4. Logical Issues:**
- Scoring edge cases: What happens with zero reviewers? Division by zero in weighted averages? Missing scores for some criteria?
- Review state machine: Can a review be submitted twice? Can a finalized review be re-opened? Are state transitions validated?
- Goal completion: Can goal progress exceed 100%? Is negative progress possible?
- Cron timing: Are review cycle deadlines correctly calculated? Timezone handling?
- Alert thresholds: Are alert conditions evaluated consistently? Can stale alerts fire repeatedly?

Rate every finding: Critical / High / Medium / Low. Include file:line references and a concrete fix recommendation for each.
  </action>
