<objective>
Audit the Performance module's REST endpoints, admin pages, and module orchestrator for security, performance, duplication, and logical issues. This plan covers the endpoint/UI layer: REST API permission callbacks, admin page rendering, nonce validation, and the module bootstrap.

Purpose: REST endpoints and admin pages are the attack surface for the Performance module. Missing permission_callback or nonce checks on review submission/scoring endpoints would allow unauthorized performance data manipulation. The admin pages (1,671 lines) render performance data that may contain unescaped employee information. The justification workflow -- where managers provide written justifications for scores -- must prevent employees from reading their own scores before formal publication.
Output: `.planning/phases/07-performance-audit/07-02-performance-admin-rest-findings.md`
</objective>

<task type="auto">
  <name>Task 1: Read and audit Performance REST, Admin, and Module orchestrator</name>
  <files>
    includes/Modules/Performance/Rest/Performance_Rest.php (730 lines -- REST API endpoints for reviews, scores, goals)
    includes/Modules/Performance/Admin/Admin_Pages.php (1671 lines -- admin UI rendering, AJAX handlers, form processing)
    includes/Modules/Performance/PerformanceModule.php (234 lines -- module bootstrap, hook registration, table definitions)
  </files>
  <action>
Read all 3 PHP files. Audit each file against the 4 metrics:

**1. Security (Critical focus):**

For Performance_Rest.php:
- List every `register_rest_route()` call. For each endpoint:
  - What is the `permission_callback`? Is it `__return_true` (Critical -- unauthenticated access)?
  - Does it check a specific capability like `sfs_hr.manage` or `sfs_hr.view`?
  - Are write endpoints (POST/PUT/DELETE) protected with appropriate capabilities?
- Can an employee access GET endpoints that return other employees' scores or justifications?
- Is there a score-viewing endpoint that should restrict access until scores are "published"?
- `sanitize_callback` and `validate_callback` on route args: are they present?

For Admin_Pages.php:
- Every `$_POST`, `$_GET`, `$_REQUEST` usage: Is nonce checked before processing?
- Every output of employee data: Is `esc_html()`, `esc_attr()`, `wp_kses()` applied?
- AJAX handlers: Do they verify `check_ajax_referer()` or `wp_verify_nonce()`?
- Capability checks: Does every admin page callback check `current_user_can()` before rendering?

For PerformanceModule.php:
- Hook registrations: Are any hooks registered without appropriate access control?
- Table definitions: Are CREATE TABLE statements using correct column types and indices?
- Does the module register any public-facing shortcodes or actions that should be restricted?

**2. Performance:**
- Admin_Pages.php: Does admin list rendering query all reviews without pagination?
- REST endpoints: Are list endpoints paginated? Do they have sensible LIMIT defaults?
- Are there dashboard widgets or summary queries that do full-table scans?
- Does admin page rendering trigger multiple queries that could be combined?

**3. Duplication:**
- Is permission checking logic repeated across REST callbacks vs admin page checks?
- Are there repeated data formatting/transformation patterns between admin pages and REST responses?
- Does PerformanceModule.php duplicate initialization logic found in the services?

**4. Logical Issues (Justification workflow focus):**
- Justification access control: Can an employee view the justification text written about them before the manager formally publishes/shares it?
- Score visibility timeline: Is there a "draft" vs "published" state for reviews? Can employees see draft scores?
- Manager vs HR access: Can a department manager see scores for employees outside their department?
- Admin page state: Can admin actions (approve, reject, finalize) be performed out of order?
- REST endpoint idempotency: Can the same score be submitted multiple times, overwriting previous data?

Rate every finding: Critical / High / Medium / Low. Include file:line references and a concrete fix recommendation for each.
