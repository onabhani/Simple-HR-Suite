<task type="auto">
  <name>Task 1: Read and audit My Profile page and EmployeesModule orchestrator</name>
  <files>
    includes/Modules/Employees/Admin/class-my-profile-page.php (1,160 lines -- self-service employee profile with tabs: overview, assets, leave, etc.)
    includes/Modules/Employees/EmployeesModule.php (23 lines -- module bootstrap, admin-only hook registration)
  </files>
  <action>
Read both PHP files. Audit against the 4 metrics:

**EmployeesModule.php (23 lines):**
- Bootstrap check: Does it use bare ALTER TABLE or unprepared SHOW TABLES? (Critical antipattern from Phase 04/08)
- Hook registration: Does it do any DB operations on every admin load?
- File inclusion: Are the require_once paths safe? Any conditional loading issues?

**class-my-profile-page.php (1,160 lines):**

**1. Security (Critical focus):**
- Catalogue every `$wpdb` call (all 31): Is `$wpdb->prepare()` used? Flag any raw interpolation as Critical.
- Data scoping: The menu uses `read` capability (any logged-in user). Does every query filter by `user_id = get_current_user_id()`? Can an employee see another employee's data by manipulating parameters?
- IDOR check: Are there any employee_id parameters from GET/POST that bypass the user_id scoping? (e.g., `$_GET['employee_id']` used instead of the user_id-derived employee record)
- Output escaping: Are all employee fields escaped with `esc_html()`, `esc_attr()` when rendered?
- Tab rendering: Does the `$active_tab` from `$_GET['tab']` (line 61, sanitize_key used) get used safely, or could it include arbitrary files?
- Write operations: Does My Profile allow any self-service write operations (edit own profile, submit requests)? If so, are nonce and ownership checks present?

**2. Performance:**
- Tab data loading: Does render_page() load ALL tab data regardless of which tab is active? Or does it lazy-load per-tab?
- N+1 patterns: Are there queries inside loops for assets, leave, attendance data?
- Unbounded queries: Any SELECT without LIMIT for employee's assets, leave history, etc.?

**3. Duplication:**
- Code shared with Employee Profile page: Are similar rendering patterns, date formatting, or query patterns duplicated between the two files?
- Tab rendering: Is there repeated tab header/content rendering logic?

**4. Logical issues:**
- Dual hire_date/hired_at: Lines 218-219 use same fallback pattern as Employee Profile (line 175). Document whether both files are consistent.
- Employee record linkage: Line 49-54 queries by user_id -- what happens if user_id maps to no employee? Is the error handling adequate?
- Tab state: Is there any state leakage between tabs?

For each finding, assign severity (Critical/High/Medium/Low) and provide a concrete fix recommendation.
