  <action>
Read every PHP file listed above. For each file, check against these audit metrics and record every finding:

**Security — XSS / Output Escaping (highest priority for Frontend):**
- Every `echo`, `printf`, string interpolation inside HTML output — is the value escaped with `esc_html()`, `esc_attr()`, `esc_url()`, or `wp_kses()`? Flag unescaped dynamic output as High (or Critical if it outputs user-controlled data like names, emails, or form fields).
- Special attention to tab render methods — these output large HTML blocks. Check EVERY variable interpolated into HTML.
- Check `wp_localize_script()` and inline JS — are PHP variables JSON-encoded properly or injected raw?

**Security — Auth and Access Control:**
- Every shortcode callback in `Shortcodes.php` — does it check `is_user_logged_in()` and/or `current_user_can()` before rendering sensitive data? A shortcode that renders employee data without auth is Critical.
- Tab access control — does `Tab_Dispatcher.php` or `Navigation.php` enforce role-based visibility? Can a user access a tab they should not see by manipulating the URL/parameter?
- `Role_Resolver.php` — does the role resolution logic have any bypass paths?
- Cross-employee data access — can Employee A see Employee B's data through a tab by manipulating an employee_id parameter?

**Security — Nonce Validation:**
- Any form in tab renderers that submits data — does the handler verify a nonce? Flag missing nonce checks as High.
- AJAX calls initiated from frontend tabs — are they protected with nonces?

**Security — SQL Injection:**
- Any `$wpdb` queries in Frontend/ files (tab renderers sometimes query directly) — are they prepared? Flag raw interpolation as Critical.

**Performance:**
- Queries inside render methods that run on every page load — especially in tabs that may not be visible. Flag as Medium.
- N+1 patterns: loops in tab renderers that query per-row. Flag as High.
- Large unbounded queries in dashboard/overview tabs. Flag as Medium.

**Duplication and Logic:**
- Repeated HTML/rendering patterns across tab files that could be shared components. Flag as Low.
- Inconsistent auth checking patterns (some tabs check, others do not). Flag as Medium.
- Dead tabs or unreachable code paths. Flag as Low.

For each finding, record:
1. **Severity**: Critical / High / Medium / Low
2. **Category**: Security-XSS, Security-Auth, Security-Nonce, Security-SQLi, Performance, Duplication, or Logic
3. **File and line number(s)**: e.g., `includes/Frontend/Tabs/LeaveTab.php:L145`
4. **Description**: What the issue is
5. **Evidence**: The problematic code snippet (1-5 lines)
6. **Fix recommendation**: Concrete fix

IMPORTANT: Start with `Shortcodes.php` (3.8K lines) as it contains all shortcode entry points and is the primary attack surface. Then audit tab files largest to smallest. Finally audit the infrastructure files (Navigation, Tab_Dispatcher, Role_Resolver, reminders).
