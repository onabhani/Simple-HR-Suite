  <action>
Read every PHP file listed above. For each file, check against the following 4 audit metrics and record every finding:

**CORE-01 — Security (Critical/High priority):**
- SQL injection: Every `$wpdb->query()`, `$wpdb->get_results()`, `$wpdb->get_var()`, `$wpdb->get_row()` call — is the query built with `$wpdb->prepare()` or is user input concatenated/interpolated directly? Flag any raw interpolation as Critical.
- Auth bypass: Every AJAX handler (`wp_ajax_*`), REST endpoint callback, and admin page render — does it check `current_user_can()` with an appropriate capability before performing the action? Flag missing checks as High.
- Nonce validation: Every form submission handler and AJAX callback — does it verify a nonce with `wp_verify_nonce()` or `check_ajax_referer()`? Flag missing nonce checks as High.
- Data sanitization: Is user input from `$_POST`, `$_GET`, `$_REQUEST` sanitized with `sanitize_text_field()`, `intval()`, `absint()`, etc. before use? Flag unsanitized input as High.
- Output escaping: Is data echoed to HTML escaped with `esc_html()`, `esc_attr()`, `esc_url()`, `wp_kses()`? Flag unescaped output as Medium.

**CORE-02 — Performance (Medium/High priority):**
- N+1 queries: Loops that execute a DB query per iteration instead of a single batch query. Flag as High.
- Unbounded queries: `SELECT` without `LIMIT` on tables that could grow large (employees, sessions, audit logs). Flag as Medium.
- Heavy admin_init: Code that runs on every admin page load but only needs to run on specific pages. Flag as Medium.
- Unnecessary option autoloading: Large serialized data stored with `autoload=yes`. Flag as Low.
- Redundant DB calls: Same query executed multiple times without caching the result. Flag as Medium.

**CORE-03 — Duplication and Logical Issues (Medium/Low priority):**
- Duplicated logic: Similar code blocks across multiple files (e.g., employee lookup, permission checks, date formatting). Note the pattern and suggest a shared helper.
- Dead code: Functions/methods that are never called from anywhere in the codebase. Flag as Low.
- Race conditions: Operations that read-then-write without locking (e.g., balance updates, counter increments). Flag as High if financial, Medium otherwise.
- Incorrect logic: Wrong operator, off-by-one errors, missing edge cases. Flag by impact severity.
- Inconsistent patterns: Some files use one approach while others use another for the same task. Flag as Low.

For each finding, record:
1. **Severity**: Critical / High / Medium / Low
2. **Category**: Security, Performance, Duplication, or Logic
3. **File and line number(s)**: e.g., `includes/Core/Admin.php:L245-L260`
4. **Description**: What the issue is, in one sentence
5. **Evidence**: The problematic code snippet (keep short, 1-5 lines)
6. **Fix recommendation**: Concrete fix in one sentence

IMPORTANT: Start with `Admin.php` (7K lines) — it will have the most findings. Read it in chunks if needed. Then proceed through remaining files largest to smallest.
