    includes/Modules/Resignation/Admin/Views/class-resignation-settings.php (136 lines -- resignation settings page)
  </files>
  <action>
Read all 4 files completely. Audit against the 4 metrics:

**1. Security (Critical focus):**
- Catalogue every `$wpdb` call across all 4 files: Is `$wpdb->prepare()` used? Flag any raw string interpolation as Critical.
- ResignationModule.php install() DDL: Does it use bare ALTER TABLE or unprepared SHOW TABLES / information_schema? (Critical antipattern from Phase 04, 08, 11)
- class-resignation-list.php: Check all status-change actions (approve, reject, cancel) for `current_user_can()` capability guards at entry. Are nonces validated with `check_admin_referer()` or `wp_verify_nonce()`?
- class-resignation-list.php: Does the list view filter by department for non-admin users, or does every admin page viewer see all-org resignation data?
- class-resignation-settings.php: Does saving settings require `current_user_can('manage_options')` or equivalent? Is nonce checked?
- Output escaping: Are all echoed values wrapped in `esc_html()`, `esc_attr()`, or similar? Flag unescaped `$_GET`/`$_POST`/`$_REQUEST` or DB values in HTML output.

**2. Performance:**
- class-resignation-list.php: Is the resignation list query paginated with LIMIT/OFFSET? Or does it load all resignations? If no pagination, flag as High.
- Are there N+1 patterns (e.g., loading employee name per resignation row in a loop instead of JOIN)?
- Are there any `information_schema` queries? (antipattern from prior phases)

**3. Duplication:**
- Does class-resignation-list.php duplicate filtering/query logic that exists in class-resignation-service.php?
- Are status badge rendering or action button patterns duplicated across views?

**4. Logical issues:**
- Resignation list status filter: Can admin view filter by any status, or are there hidden/invalid statuses that slip through?
- Settings: Are setting keys properly prefixed with `sfs_hr_`? Are defaults sensible?
- Does the admin router properly gate subpages?

For each finding, assign severity (Critical/High/Medium/Low) and provide a concrete fix recommendation.
