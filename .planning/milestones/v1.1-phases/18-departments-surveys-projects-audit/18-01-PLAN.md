  <action>
Read all 3 files completely. Audit against the 4 metrics:

**1. Security (Critical focus -- manager assignment auth and survey data isolation):**

DepartmentsModule.php:
- Check for bare ALTER TABLE, unprepared SHOW TABLES, information_schema queries (antipattern from Phase 04/08/11/16).
- Department manager assignment: Does handle_save/update require sfs_hr.manage capability? Can any authenticated user set themselves as manager_user_id or hr_responsible_user_id?
- Are all $wpdb queries using $wpdb->prepare()? Catalogue every raw interpolation.
- REST endpoints: Check permission_callback on every register_rest_route. Any __return_true callbacks? Any routes returning data without department scoping?
- Nonce verification: Are all POST handlers using check_admin_referer() or wp_verify_nonce()?

SurveysModule.php:
- Check for bare ALTER TABLE, unprepared SHOW TABLES, information_schema queries.
- Survey response submission: Can Employee A submit a response as Employee B? Is employee_id derived from get_current_user_id() or from POST parameters?
- Survey response retrieval: Can Employee A view Employee B's responses? Is there proper ownership/scoping?
- Are all $wpdb queries using $wpdb->prepare()?
- REST endpoints: Check permission_callback. Any __return_true? Can unauthenticated users submit survey responses?
- Admin survey management: What capability is required to create/edit/delete surveys?

SurveysTab.php:
- Does the frontend tab check that the current user is viewing only their own survey data?
- Is all output escaped with esc_html(), esc_attr(), wp_kses()?
- Any inline $wpdb queries that should be delegated to the service layer?

**2. Performance:**
- N+1 query patterns: Does department listing or survey response retrieval query inside loops?
- Unbounded queries: Are there any SELECT without LIMIT on potentially large tables?
- Heavy admin_init hooks: Any expensive operations on every admin page load?

**3. Duplication:**
- Does DepartmentsModule duplicate org-chart or employee-listing logic found in Core/ or Employees module?
- Does SurveysModule duplicate notification dispatch logic that exists in Core/Notifications?
- Any repeated SQL patterns that could use a shared helper?

**4. Logical Issues:**
- Department deletion: What happens to employees assigned to a deleted department? Is there orphan prevention?
- Survey lifecycle: Can responses be submitted after a survey is closed/archived?
- Manager cascading: If a department manager is changed, are the old manager's dynamic capabilities revoked?
- Race conditions: Any check-then-act patterns without DB-level guards?
